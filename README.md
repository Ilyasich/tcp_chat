# tcp_chat

Простая реализация TCP-чата на Go (сервер + клиент).

Проект предназначен для изучения простого механизма обмена сообщениями через TCP.

Структура проекта:

- `cmd/` — точка входа для сервера (запуск сервера: `go run ./cmd`).
- `server/` — реализация TCP-сервера.
- `client/` — пример простого TCP-клиента.
- `config/` — общая логика чата (broadcaster, обработка подключений).

Как запустить

1. Собрать/запустить сервер:

```bash
cd /path/to/tcp_chat
go run ./cmd
```

Сервер слушает на `localhost:8000` по умолчанию.

2. Запустить клиент (в другом терминале):

```bash
cd /path/to/tcp_chat
go run ./client
```

Клиент при подключении попросит ввести никнейм, после чего все введённые строки отправляются на сервер и рассылаются всем подключённым клиентам.

Короткий пример (отправка данных из одного вызова):

```bash
echo -e "alice\nhello from alice" | go run ./client
```

Замечания и улучшения

- Логер использует `go.uber.org/zap`.
- Основные потоки: `Broadcaster` поддерживает список подключённых клиентов и пересылает сообщения.
- В текущей реализации используется простая передача сообщений и закрытие каналов при выходе клиента. Это соответствует учебной реализации, но в высоконагруженном приложении стоит предусмотреть дополнительные проверки/синхронизацию и аккуратное завершение горутин.
